# Testing Requirements

- Unit Tests — services, prompt builders, utils
- API Tests — API routes / Server Actions, including auth and validation failure paths
- UI Tests — key components (lead form, pipeline board, chat interface)
- Integration Tests — end-to-end flows (create lead → research → outreach; interested → meeting → calendar event; proposal → export)

Each phase's completion should include tests for that phase's new functionality before moving to the next phase.
